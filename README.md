# 学校のプロジェクトのやつ

## 概要
複数校をまたぐプロジェクト運営
プロジェクトを創造・運営・引継ぎ・死んだら発掘
プロジェクトの中に概要・会話チャットやリポジトリなどのurl/メモ/ハッシュタグ用意
メモの中には新規機能提案・バグや脆弱性の改善など数種類用意しその中で原因や結果をまとめて送るチャットを用意
discordなどの会話のノイズをなくし少しの文で理解できる
ある程度機能やバグの原因や修正方法がわかったらそれらを一つの文にまとめて昇華する
ハッシュタグにより興味がありそうなプロジェクトをユーザーに提案し死んだときも残り続き誰かが（プロジェクト作成者の学校に限らない）見つけて続けてくれるかもしれないし、続けなくても誰かがそのプロジェクトを確認しそれを糧に新たな分野への開拓につながるかもしれないそのようなアプリ
## プロジェクトdb構造
プロジェクト

- projectid
- 概要(テキスト)
- メイン会話チャットurl(txt)
- メインリポジトリurl(txt)
- その他のurl(json)
- ハッシュタグ(json)
- メモid

## メモdb構造
メモ

- id
- 種類(新規機能提案・バグなどの)(number)
- その他の種類（煩雑になりできるだけ使わないでほしい）(種類は0の時)(txt)
- メモテキスト(txt)
- 会話ーまだ実装しない
